# 自媒体内容模型构建器

## 快速启动（3 步）

时间紧？先走快速路径，再回来深入：

```
Step 1: 定方向 (5分钟)
  python scripts/validate_content_model.py --quick
  -> 回答 5 个问题，快速筛选方向是否可行

Step 2: 建模型 (15分钟)
  阅读 references/model-building-methodology.md 的 4+1 要素章节
  填写五要素文档模板，完成内容模型

Step 3: 快验证 (10分钟)
  手动搜 3-5 个关键词，运行完整模式:
  python scripts/validate_content_model.py
  或直接进入发帖测试:
  python scripts/design_experiment.py
```

> 快速模式不能替代完整验证，但能帮你快速排除明显不可行的方向。

## 核心哲学

这个工具与常见的自媒体工具不同：
- **不追热点** — 热点是红海竞争，第一性原理关注的是永恒的信息需求
- **不采集爆款** — 模仿他人的内容模型天然滞后，我们要构建自己的模型
- **不盲目测试** — 先对照验证（用平台已有内容验证假设），再发帖测试（精确变量测试）

### 四个设计原则

1. **回归第一性原理** — 自媒体本质是信息的生产与分发，依据人性论、平台分发机制、创作本质构建模型
2. **模型构建而非模型采集** — 形成自己的内容模型和运营模式，获得先发优势
3. **最小可行性验证** — 测试期内容做轻，不在试探期浪费资源
4. **验证前置，避免盲试** — 模型先在平台已有内容中对照验证，通过后才能进入发帖验证

### 反幻觉规则（重要）

**这个 skill 的使用过程中，Codex 必须严格遵守以下规则，否则会输出虚假数据误导用户：**

1. **禁止编造平台数据** — 不得杜撰"搜索频率上升""某人群占比 X%""平台趋势变化"等数据。所有平台数据必须来自用户手动搜索后的输入
2. **禁止跳过用户操作步骤** — 每个要求用户"搜索""输入""记录"的步骤，Codex **必须等待用户完成操作**后才继续推进，不得代替用户完成
3. **禁止假设用户已知信息** — 不要假设"用户应该知道自己的数据"。用户可能没有精确的数字，引导用户给估算值即可
4. **明确区分类比和事实** — 如果使用行业通用数据做参考（如"小红书收藏/点赞比通常在 0.3~0.7"），必须标明这是经验参考，不是用户账号的实际数据
5. **当不确定时，要求用户确认** — 任何你认为"可能知道也可能不知道"的信息，默认用户不知道，主动询问

**违反这些规则的后果**：输出虚假分析结论，用户做出错误决策，浪费时间和资源。这不是小事。

## 工作流程

```
探索定位 -> 模型构建 -> 对照验证 -> 发帖测试 -> 信号分析 -> 迭代
```

### 阶段 1：探索与定位

帮助用户梳理自身资源、选择信息价值方向、定义目标人群。

- 阅读 `references/first-principles.md` 理解信息供需第一性原理
- 阅读 `references/model-building-methodology.md` 的"探索与定位"部分
- 小红书用户同时阅读 `references/xiaohongshu-platform.md` 了解平台生态

### 阶段 2：内容模型构建

将账号定位转化为基于第一性原理推导出的 4+1 要素内容模型。

从核心公式 `信息传递效率 = (信息价值 × 信息可接受度) / 信息获取成本` 推导出四个核心要素，从互动概率公式推导出一个从属要素：

**核心四要素（从信息传递效率公式推导）：**

1. **信息价值定位** — 提供什么信息？从六种永恒信息需求中选择主方向（怎么做/选哪个/为什么/跟我一样/新东西/好看的）
2. **目标人群画像** — 谁最需要这个信息？从信息缺口出发定义人群，而非人口统计标签
3. **内容形式体系** — 什么形式让信息最容易被理解？载体、结构、视觉风格、语言风格
4. **获取策略** — 如何降低用户的信息获取成本？标题、封面、开头钩子的设计

**一个从属要素（从互动概率公式推导）：**

5. **互动设计** — 如何自然引导用户从消费转向参与？收藏诱因、评论触发、分享动机

> 互动就是信息价值最好的佐证，**互动设计是前四个要素的自然延伸**，不能脱离信息价值独立存在。

**运营规划（独立于内容模型）：**

- **可持续性机制** — 信息输入源、内容生产流程、迭代空间——这些很重要，但属于运营执行层面，不是内容模型的第一性原理

详细定义参考 `references/model-building-methodology.md` 的"4+1要素"章节。
使用内容模型文档模板记录结果（见同文件的"内容模型文档模板"）。

### 阶段 3：对照验证（平台已有内容）

**不发帖，用平台上已有的内容检验模型假设。**

这是 MVT 的第一阶段，也是最重要的阶段 — 零成本验证需求是否存在。

流程：
1. 将模型假设转化为可验证预测
2. 手动在小红书搜索相关关键词，记录观察数据
3. 运行 `scripts/validate_content_model.py` 输入观察数据
4. 获取置信度评估和是否进入发帖测试的建议

详细验证框架参考 `references/mvt-framework.md` 的"阶段一：对照验证"。

### 阶段 4：发帖测试设计

只有对照验证通过后才进入发帖测试。

核心原则：**一次只测一个变量**。

- 运行 `scripts/design_experiment.py` 设计实验
- 预设信号灯（止损/加注/观察）
- 输出实验方案到 Markdown 文件

### 阶段 5：信号跟踪与决策

发帖后跟踪数据，根据信号灯规则决策。

- 运行 `scripts/analyze_post_signals.py` 输入帖子数据
- 获取信号灯状态和综合判断（止损/加注/观察）
- 根据建议决定行动

|------|------|------|
| `scripts/validate_content_model.py` | `python scripts/validate_content_model.py` | 发帖前对照验证；支持交互式输入和 CSV 导入。加 `--quick` 进入 5 题快速定性评估 |

## 使用示例

### 示例 1：从零开始搭建账号

```
用户: "我想做小红书的职场内容，但不知道从哪开始"

你的工作流程:
1. 阅读 first-principles.md 理解信息供需（阶段 1）
2. 阅读 xiaohongshu-platform.md 了解平台（阶段 1）
3. 引导用户确定信息价值定位（"怎么做"类：职场效率教程）
4. 构建 4+1 要素模型（阶段 2）
5. 引导用户在平台搜索"职场效率""新手办公"等关键词
6. 运行 validate_content_model.py 输入观察数据（阶段 3）
7. 根据验证结果决定是否进入发帖测试
```

### 示例 2：已有账号增长停滞

```
用户: "我发了 20 篇了数据一直上不去"

你的工作流程:
1. 让用户整理 20 篇的数据
2. 运行 analyze_post_signals.py 分析信号
3. 如果红灯亮了 -> 回阶段 1/2 重新定位
4. 如果是黄灯 -> 检查内容模型一致性
5. 如果是数据波动 -> 增加样本量
```

### 示例 3：想验证一个内容方向

```
用户: "我想试试做'职场情绪管理'这个方向"

你的工作流程:
1. 帮助用户建立内容模型假设（4+1 要素）
2. 将假设转化为可验证预测
3. 在小红书搜索"职场情绪""职场焦虑"等关键词
4. 运行 validate_content_model.py
5. 如果置信度 >= 0.6 -> 设计发帖测试
6. 如果置信度 < 0.35 -> 建议换个方向
```
