## 核心理念

### 为什么要"最小可行性"？

```
验证成本 = 内容生产成本 + 时间成本 + 账号风险成本
```

**最小可行性验证的核心原则**：
1. **先对照、后测试** —— 能用平台已有内容验证的，绝不发帖验证
2. **一次只测一个变量** —— 否则你无法知道什么在起作用
3. **预设信号灯** —— 在开始前就定义好"什么算成功，什么算失败"
4. **轻量启动** —— 试探期投入最低成本

### 验证的目标不是"成功"

**验证的唯一目标是"获取可靠信息"**。
- 如果模型有效 -> 你知道了该加注什么
- 如果模型无效 -> 你知道了该止损什么
- 两种都是有效验证

## 阶段一：对照验证（平台已有内容）

### 核心思路

**不需要自己发帖，用平台上已有的内容来检验你的模型假设。**

如果同类（或同方向）的内容在平台上已经有高互动表现，说明需求存在；
如果没有，说明要么需求不存在，要么你的表达方式有问题。

### 对照验证流程

#### 步骤 1：提取模型假设的可验证预测

将内容模型的 4+1 要素转化为可验证的预测：

| 要素 | 可验证预测 |
|------|-----------|
| 信息价值定位 | "标题包含[关键词]的内容互动率高于平均水平" |
| 目标人群 | "面向[人群]的内容有稳定的互动" |
| 内容形式 | "采用[结构]的内容收藏率比其他形式高" |
| 互动设计 | "包含[互动引导]的内容评论数更多" |
| 可持续性 | "高频更新[方向]的账号粉丝增长更快" |

**示例**：

```
假设："对于职场新人，'如何做'类的效率工具教程在小红书上有稳定需求"

可验证预测：
1. 搜索"职场效率""新手办公"等关键词有大量高互动笔记
2. 标题含"效率""工具""技巧"的笔记收藏率高于平台均值
3. 清单体结构的教程类笔记互动 > 故事体教程
```

#### 步骤 2：在平台上搜集证据

**操作方法**（人工，非自动化）：

1. **搜索验证**：
   - 在小红书搜索你的核心关键词
   - 记录前 20 篇高互动笔记的：互动数据、内容形式、标题结构
   - 判断这些内容是否符合你的模型假设

2. **竞品账号分析**：
   - 找到 3-5 个和你方向相似的账号
   - 分析他们的内容模型的 4+1 要素
   - 记录他们的粉丝量、互动率、更新频率

3. **内容模式识别**：
   - 这些高互动内容共有的模式是什么？
   - 你的模型中哪些被验证了，哪些没有被验证？

#### 步骤 3：使用验证脚本量化分析

运行 `scripts/validate_content_model.py` 脚本：
- 输入：你从平台上采集的对照数据（格式见脚本说明）
- 输出：你的模型假设被支持的置信度评估
- 决策：是否进入阶段二

### 对照验证的判断标准

| 对照结果 | 判断 | 行动 |
|---------|------|------|
| 同类内容互动率高，且你的模型有差异化 | ✅ 强支持 | 进入发帖测试 |
| 同类内容互动率高，但你的模型无差异化 | ⚠️ 弱支持 | 优化差异化，再对照验证 |
| 同类内容互动率低，但需求持续性存在 | 🔄 中性 | 考虑表达方式是否合适 |
| 同类内容互动率低，且无明显需求 | ❌ 不支持 | 调整或放弃该方向 |

## 阶段二：发帖测试（精确变量测试）

### 核心原则

只有阶段一通过后，才进入发帖测试。

**"用发帖验证，而不是用发帖探索。"**

### 实验设计方法

#### 单一变量测试

每次发帖只改变一个变量，其他变量保持不变：

```
控制组：标题A + 封面A + 结构A
测试组：标题B + 封面A + 结构A
→ 差异 = 标题的影响
```

**可测试的变量**：
| 变量 | 测试方法 | 最少样本量 |
|------|---------|-----------|
| 标题公式 | 同一内容换不同标题 | 3-5篇 |
| 封面风格 | 同一内容换不同封面 | 3-5篇 |
| 内容结构 | 同一信息用不同结构 | 3-5篇 |
| 发布时间 | 同一内容不同时间发 | 2-3次 |
| 互动引导 | 有无互动引导的比较 | 3-5篇 |

#### A/B 测试设计

```
┌─────────────────────────────────────────┐
│ 实验名称：[测试变量]                      │
│ 假设：[具体可测量假设]                    │
│                                         │
│ 控制组条件：[不变的部分]                  │
│ 测试组条件：[变化的变量]                  │
│                                         │
│ 成功标准（加注信号）：                    │
│ - 测试组互动率 > 控制组 50%以上          │
│ - 测试组收藏率 > 控制组 30%以上          │
│                                         │
│ 失败标准（止损信号）：                    │
│ - 测试组互动率 ≤ 控制组                  │
│ - 测试组点击率低于平台该类内容中位数       │
│                                         │
│ 测试周期：[天数]                         │
│ 单组样本量：3-5篇                       │
└─────────────────────────────────────────┘
```

## 信号灯止损/加注系统

### 信号灯定义

在开始测试前预设清晰的标准：

#### 🔴 红灯（止损）

| 信号 | 判定标准 | 行动 |
|------|---------|------|
| 点击率低于预期 | 连续 5 篇点击率 < 平台同类中位数 | 停止该方向，回阶段一 |
| 互动率持续走低 | 连续 5 篇互动率递减 | 暂停发帖，重新验证模型 |
| 收藏率归零 | 连续 3 篇收藏为 0 | 重新评估信息价值定位 |
| 负反馈（取关/举报） | 负反馈率 > 5% | 立即暂停，检查内容 |

#### 🟢 绿灯（加注）

| 信号 | 判定标准 | 行动 |
|------|---------|------|
| 互动率持续上升 | 连续 5 篇互动率递增 | 增加发布频率 |
| 收藏率超预期 | 收藏率 > 平台同类 top 25% | 深化该方向内容 |
| 粉丝增长加速 | 粉丝增长曲线上升 | 投入更多资源 |
| 搜索流量占比高 | 搜索来源流量 > 30% | 强化 SEO 策略 |

#### 🟡 黄灯（观察）

| 信号 | 判定标准 | 行动 |
|------|---------|------|
| 数据波动大 | 单篇数据起伏不定 | 增加样本量 |
| 互动分散 | 没有稳定的互动模式 | 检查内容模型的一致性 |
| 单篇爆款但无持续 | 爆款后回归平庸 | 测试可复制性 |

### 信号灯使用纪律

1. **预设不可变**：信号灯在实验开始前设定，不可在结果出来后修改
2. **信号不撒谎**：如果红灯亮了，接受它，不要找借口
3. **组合判断**：单一信号不足以决策，看信号组合
4. **时间边界**：设定明确的测试周期，到时间就判断，不要无限延长

## 验证记录与迭代

### 每轮验证必须记录

```markdown
## 验证轮次：[编号]

### 验证类型
[ ] 对照验证（平台已有内容）
[ ] 发帖测试（精确变量测试）

### 模型假设
[假设描述]

### 验证设计
[测试了什么变量/验证了什么预测]

### 结果数据
[关键数据汇总]

### 信号灯判断
[🔴/🟡/🟢]

### 决策行动
[止损/调整/加注]

### 学到的教训
[这个验证给你的最大认知是什么]
```

### 迭代原则

- **每轮验证只聚焦一个核心问题**
- **根据验证结果调整模型，而不是调整验证标准**
- **连续 2 轮红灯 = 该方向暂时放弃**
- **连续 3 轮绿灯 = 模型进入稳定运营期**，降低验证频率
