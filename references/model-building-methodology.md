## 核心概念：什么是内容模型

**内容模型 ≠ 内容模板。**

内容模型是一套关于"你的账号生产什么信息、以什么形式、给谁看、为什么他们会互动、如何持续复制"的可重复系统。

| 要素 | 模板 | 模型 |
|------|------|------|
| 关注点 | "长什么样" | "为什么有效" |
| 可复制性 | 机械复制 | 原理复用 |
| 迭代方式 | 换皮 | 优化参数 |
| 竞争壁垒 | 无（人人可抄） | 有（理解成本高） |
| 与账号关系 | 单篇内容 | 整个账号的DNA |

## 从模型采集到模型构建的思维转变

### 常见误区：采集他人的内容模型

大多数自媒体工具的做法：
1. 爬取爆款内容
2. 分析标题公式、封面风格、关键词
3. 推荐用户"照这个方向做"

**问题**：
- 等你分析出来并开始做，同质化已经严重
- 你永远在追赶，而不是在定义
- **你模仿的是"结果"，不是"原因"**

### 正确路径：构建自己的内容模型

```
不是 "别人做了什么 → 我也做"
而是 "用户需要什么 → 我能提供什么 → 以什么形式 → 如何持续验证"
```

### 模型构建 vs 模型采集的对比

| 维度 | 模型采集（跟随） | 模型构建（原创） |
|------|-----------------|-----------------|
| 起点 | 别人的爆款 | 用户的信息需求 |
| 差异化 | 在形式上微调 | 在价值上深挖 |
| 可持续性 | 低（需要不断寻找新爆款） | 高（不断深化一个方向） |
| 账号资产积累 | 慢（用户因单篇内容关注） | 快（用户因价值定位关注） |
| 平台推荐效率 | 低（算法难以定位账号） | 高（算法明确知道推荐给谁） |

## 内容模型的 4+1 要素

一个完整的内容模型由四个核心要素（从第一性原理公式 `信息传递效率 = 信息价值 × 信息可接受度 / 信息获取成本` 推导）和一个从属要素（从互动概率公式推导）构成。

```
核心四要素：
  1. 信息价值定位     → 信息价值
  2. 目标人群画像     → 信息价值的受体
  3. 内容形式体系     → 信息可接受度
  4. 获取策略         → 信息获取成本（新增）

从属要素：
  5. 互动设计         → 互动收益 - 互动成本
```

> **互动设计是从属的**：如果你的信息价值定位精准、人群对路、形式合适、获取成本低，互动会自然发生。互动设计的作用是放大和引导，不是创造本来不存在的互动理由。

### 1. 信息价值定位（"提供什么"）

**核心问题**：你的内容在满足用户的什么信息需求？

从第一性原理的六种永恒需求中选择：

| 需求类型 | 内容形式示例 | 信息增量来源 |
|---------|------------|------------|
| "怎么做" | 教程、攻略、方法论 | 你的经验/专业/方法论 |
| "选哪个" | 测评、对比、推荐 | 你的使用体验/分析框架 |
| "为什么" | 解读、分析、底层逻辑 | 你的思考深度/行业洞察 |
| "跟我一样" | 共鸣、日常、身份表达 | 你的真实体验/价值观 |
| "新东西" | 资讯、趋势、发现 | 你的信息渠道/筛选能力 |
| "好看的" | 视觉内容、审美分享 | 你的审美/创作能力 |

**关键原则**：
- 选择一个**主需求**，1-2个**辅助需求**
- 不要试图满足所有需求（没有定位）
- 主需求决定了你的"赛道"

### 2. 目标人群画像（"给谁看"）

**核心问题**：谁最需要你提供的信息？

定义维度：
- **可识别特征**：年龄、城市、职业、收入（可统计）
- **行为特征**：使用什么App、看什么内容、消费什么（可观察）
- **核心痛点**：他们正面临什么困扰（可共情）
- **信息缺口**：他们不知道什么、想了解什么（可补缺）

**关键原则**：
- 人群越精准，内容越有针对性
- 但人群太小则规模不足
- 用"一个典型用户"的形象来描述

### 3. 内容形式体系（"怎么做"）

**核心问题**：信息以什么载体和什么结构呈现？

定义维度：
- **载体**：图文 / 短视频 / 长视频 / 音频 / 图文+视频
- **结构**：清单体 / 故事体 / 论证体 / 清单+故事
- **视觉风格**：色调 / 字体 / 构图 / 排版
- **语言风格**：正式 / 口语 / 专业 / 亲切

**关键原则**：
- 形式取决于：平台调性 × 信息类型 × 你的能力
- 形式可以组合，但每种形式要有"标准模块"
- **形式服务于信息传递效率**，不是反过来

### 4. 获取策略（"怎么被看到"）

**核心问题**：用户在看到你的内容时，第一眼如何判断"这个值得我点开"？

这是从第一性原理公式 `信息传递效率 = 信息价值 × 信息可接受度 / 信息获取成本` 中**直接作为分母**存在的维度，却几乎在所有自媒体方法论中被忽视。

**定义维度**：

#### 标题策略
标题是降低信息获取成本的第一步。用户在信息流中花在标题上的时间不到 1 秒。

- **好奇心缺口**：让用户知道一部分但不知道全部 → "3个方法"（知道有3个，但不知道是哪3个）
- **身份关联**：让用户觉得"这跟我有关" → "职场人必看"
- **价值承诺**：明确告诉用户看完能得到什么 → "让你效率翻倍"
- **标题公式**：核心关键词 + 场景/人群 + 情绪/价值词

#### 封面策略
封面是视觉入口，决定了用户是否从"扫过"变成"停留"。

- **信息层级**：大字标题 > 视觉焦点 > 辅助信息
- **视觉一致性**：统一的封面风格形成账号辨识度
- **平台适配**：小红书封面以文字+图片叠加为主，信息密度高

#### 开头钩子
开头决定了用户是否继续往下看。前 3 行决定了 80% 的读完率。

- **痛点共鸣**："你是不是也遇到这个问题..."
- **结果前置**："用了这个方法后，我的效率提升了 3 倍"
- **反常识**："90% 的人不知道这个功能"
- **设问引思**："你有没有想过为什么..."

**关键原则**：
- 获取策略不是"骗点击"——标题/封面如果承诺了内容无法提供的价值，用户会觉得被骗，反而损害账号权重
- **获取策略是信息价值的放大器，不是替代品**
- 不同平台对标题和封面的偏好不同（小红书偏好高信息密度的文字封面，抖音偏好情绪化的视频封面）

### 5. 互动设计（"为什么互动"）— 从属要素

**核心问题**：用户在什么节点、因为什么动机会互动？

互动是从"信息消费"到"信息参与"的跃迁。从第一性原理出发：

```
互动概率 = (互动收益 - 互动成本) / 注意力剩余
```

互动设计的本质是 **提高互动收益 + 降低互动成本**，而不是"诱导"。

**设计维度**：

- **互动收益设计**：
  - 自我表达收益 → 给用户一个表达观点的理由（"你觉得呢？"）
  - 资源获取收益 → 让收藏能带来可复用的价值（模板、清单）
  - 社交展示收益 → 让转发成为社交货币（"@你需要的朋友"）
  - 利他收益 → 让点赞成为一种"支持有用内容"的行为

- **互动成本降低**：
  - 认知成本 → 互动引导要简单直接（"评论区打1" 比 "对此你怎么看" 成本更低）
  - 操作成本 → 收藏比评论更容易，所以要设计收藏诱因优先
  - 社交风险 → 不要强迫用户表达可能被反驳的观点

**关键原则**：
- ⚠️ **互动设计是前四个要素的延伸，不能独立存在**。如果你的信息价值定位不清晰、人群不对、形式不对、获取成本高，再好的互动设计也无用
- **收藏是最有价值的互动**，因为它直接向平台信号：内容"有用"
- 不同类型的互动服务于不同的目的：收藏=有用性，评论=讨论价值，分享=社交价值，点赞=低成本认可

## 运营规划（独立于内容模型）

以下维度**不属于内容模型的第一性原理**，但它们决定了你的模型能否落地。在内容模型构建完毕后，单独制定运营规划。

### 可持续性机制（"能持续吗"）

**核心问题**：你能持续产出同质量内容吗？

设计维度：
- **信息输入源**：你的信息从哪里来（阅读/经验/采访/数据）
- **内容生产流程**：从构思到发布的标准化流程
- **质量稳定性机制**：什么情况下可以降速但不能降质
- **迭代空间**：模型允许在哪些维度上优化

**关键原则**：
- 不可持续的内容模型 = 没有模型
- 产出速度要匹配信息输入速度
- **内容模型要包含"升级路径"**——在模型内迭代，而不是推倒重来

### 平台适配

**核心问题**：同一内容模型在不同平台上需要做什么调整？

平台的分发机制差异主要在以下方面：
- **互动信号权重**：小红书看重收藏率，抖音看重完播率，B站看重评论深度
- **内容形式偏好**：小红书图文为王，抖音短视频为王，公众号长文为王
- **搜索流量占比**：小红书搜索流量占比 40-60%，其他平台搜索占比低得多
- **用户使用心智**：小红书用户"来搜索"的心智强，抖音用户"来刷"的心智强

**原则**：保持信息价值定位不变，调整获取策略和形式适配各平台。

## 模型构建流程

### 阶段一：探索与定位（1-2周）

目标：确定内容模型的方向

1. **自我资源盘点**
   - 你有什么独特经验/知识/视角？
   - 你能稳定获取什么信息？
   - 你的创作能力和时间资源？

2. **需求假设生成**
   - 从六种永恒需求中选择1-2个方向
   - 定义你打算服务的目标人群
   - 写下你的信息价值假设

3. **平台已有内容验证**
   （进入 MVT 框架的"对照验证"阶段）
   - 在平台上搜索相关内容
   - 分析高互动内容的模式
   - 验证你的假设是否有需求基础

### 阶段二：构建与验证（2-4周）

目标：搭建并测试内容模型原型

1. **设计内容模型**（4+1 要素完整文档化）
2. **最小可行性内容**（MVP内容）
   - 产出3-5篇代表模型核心的内容
   - 目的是验证模型，不是追求完美
3. **对照验证**（使用 validate_content_model.py）
4. **根据反馈迭代模型**

### 阶段三：优化与放大（4周+）

目标：验证模型的有效性并持续优化

1. **发帖测试**（使用 design_experiment.py）
2. **信号跟踪**（使用 analyze_post_signals.py）
3. **根据信号灯决策**：止损/加注/调整
4. **模型固化**：将有效的模型参数文档化

## 模型一致性自查表

内容模型最大的敌人不是"做不出来"，而是**做着做着就偏了**——今天想试这个方向，明天觉得那个也能做，两个月后回头看账号已经完全不是当初定位的样子。这就是**选题漂移**。

每次发布前，用以下清单自查一次。任何一条不符合，要么调整选题，要么调整模型。

### 常规自查（每篇内容发布前）

| # | 自查项 | 通过标准 | 不通过怎么办 |
|---|--------|---------|------------|
| 1 | 这篇内容服务的是不是模型定义的**目标人群**？ | 目标用户看完会想"这就是给我看的" | 如果不是->改选题，不要发 |
| 2 | 这篇内容的**信息价值定位**是否在模型的范围内？ | 属于模型定义的主需求或辅助需求类型 | 如果是新需求类型->先更新模型再发 |
| 3 | 这篇内容的**形式**是否符合模型的视觉和结构标准？ | 封面/标题/结构符合模型定义的形式体系 | 偶尔形式创新可以，连续3篇不符合->检查模型 |
| 4 | 这篇内容是否比上篇更**偏离模型的核心定位**？ | 与模型核心定位的方向偏差 < 30% | 连续3篇偏离->暂停发布，重新评估模型 |
| 5 | 这篇内容如果数据不好，是因为**执行问题还是模型问题**？ | 能明确归因 | 无法归因->说明变量控制不够严谨 |

### 深度自查（每周/每10篇）

| # | 自查项 | 红灯信号 |
|---|--------|---------|
| 1 | **选题来源追踪**：最近5篇选题分别来自哪里？ | 超过3篇的选题来自"看到别人做这个火了"->模型被带偏 |
| 2 | **互动模式一致性**：高互动的内容是否符合模型的互动设计预测？ | 高互动内容反而是模型没覆盖的方向->说明模型漏掉了什么 |
| 3 | **粉丝认知一致性**：新粉丝是因为什么关注你的？ | 新粉丝的理由和你模型的定位不一致->内容传达的信号错了 |
| 4 | **模型偏离度评分**：给最近10篇内容逐篇打分（完全符合模型=10分，完全偏离=0分），算平均分 | 平均分 < 7->模型需要收紧或模型本身有问题 |
| 5 | **止损触发器**：如果按模型定位持续发了一个月但数据毫无起色，是继续还是换方向？ | 1个月0起色->不是模型的问题就是赛道的问题，两者都需要重新评估 |

### 执行纪律

- 自查表不是摆设：如果自查不通过但你还是想发，说明你应该先更新模型，而不是无视自查表
- 使用 `scripts/analyze_post_signals.py` 的"发帖节奏不稳定"信号来辅助检测选题漂移——如果发帖节奏出现异常波动，往往是选题脱离模型的信号
- **一致性 > 单篇爆款**：一个一致的账号在算法中获得的长尾流量，远高于一个"偶尔爆款但定位混乱"的账号
